## 流程问题总结

1. 三步路径是正确的：第一步输出进入第二步，第二步输出进入第三步。
2. 当前主要问题是 GPTS 内置提示词尚未换成优化版，导致第一步格式不稳、第二步系统设定未统一。
3. 第三步首次运行超时后，旧代码尝试编辑不存在的上一条用户消息，导致失败。
4. 已修复代码：`edit-and-resend` 找不到可编辑用户消息时，会自动降级为重新发送。

## 下一轮测试建议

等 GPTS 内置提示词替换为 `prompts` 文件夹中的优化版后，重新运行：

```powershell
node gpts-queue.mjs --config config-test-dagang.json --reset-state --force
node gpts-queue.mjs --config config-test-adapt.json --reset-state --force
node gpts-queue.mjs --config config-test-zhengwen.json --reset-state --force
```

重点检查：

* 第一步是否严格输出 `## 原剧情细纲`，并按 8-10 条列表拆分。
* 第二步是否输出 `新世界剧情细纲`，并清除系统、积分、奖励、跨世界记忆等设定。
* 第三步正文是否不再出现系统提示、任务值、剧情偏移、进入下一世界等内容。
